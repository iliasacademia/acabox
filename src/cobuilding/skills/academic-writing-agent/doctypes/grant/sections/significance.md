# Significance Section Conventions

Significance answers "why does this work matter?" — both to the field and to the agency's mission. It is the section reviewers use to decide whether the proposal addresses a real problem.

## Principles

1. **Build a case, do not survey the field.** The Significance section is an argument, not a literature review. Each paragraph should advance the case that the proposed work is needed.

2. **Lead with the problem, not the field.** Open by stating the unsolved problem — public health burden, knowledge gap, technical bottleneck. The first sentence of Significance is doing real work; do not waste it on field-level platitudes.

3. **Cite to build the case, synthesize across citations.** Significance is citation-heavy, but list-style "Smith showed X. Jones showed Y. Lee showed Z." is weak. Synthesize: "Multiple groups have shown X under conditions Y, yet none have addressed Z." Make the citations argue for you.

4. **Three sub-arguments, typically.** Most strong Significance sections build three case elements: (a) the problem is important, (b) the current state of knowledge is insufficient, (c) progress will require what this proposal does. Use sub-headers if the proposal allows.

5. **Quantify when possible.** "Affects an estimated 2.4 million Americans" beats "is a common condition." "Current methods achieve 60% sensitivity" beats "current methods are insufficient." Specific numbers signal the author is familiar with the literature.

6. **End with the proposal's contribution.** Close Significance by previewing how this work changes the picture — without repeating the Aims or the Approach. One paragraph, claim-only.

7. **Tie to agency mission when relevant.** For NIH, public health relevance is explicit. For NSF, broader impacts. For mission-driven foundations, the foundation's stated goals. Use the agency's language where it fits naturally — do not parrot.

## Common Failure Modes

- **Significance reads as Background.** If a reader could swap your Significance into another grant on a related topic without changing it, it isn't doing argument work — it's surveying.
- **Generic statements of importance.** "This is an important area of research" tells the reviewer nothing. State *why* it is important.
- **Buried claim.** If the strongest argument for the proposal is in the third paragraph, lead with it.
- **Citation soup.** Walls of `(1-7, 12-15, 19, 21-25)` signal that the author didn't synthesize.

## Citations

Heavy citation expected. Anchor every empirical claim to a source; route any reference the author hasn't already added through Cite for verification before suggesting it. Do not invent references or attribute findings to specific authors without confirming the citation.
