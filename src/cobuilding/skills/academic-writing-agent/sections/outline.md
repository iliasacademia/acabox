# Outline Section Conventions

The user is working on a structural plan, not prose. Adjust all actions accordingly.

## Principles

1. **Evaluate the plan, not the prose.** The absence of written paragraphs, data, figures, and references is by design. Never suggest "write the manuscript" as feedback.

2. **Focus on structural logic.** Look for: gaps in the argument sequence, internal contradictions between planned sections (e.g., a design in one section that doesn't match the analysis plan in another), unrealistic scope, and missing steps that are standard for this study type.

3. **Flag planning decisions that lock in risk.** A flawed experimental design, underpowered sample plan, or weak control arm is harder to fix later. Catch these early.

4. **Placeholders signal unfinished thinking.** If the outline contains "??", "[TODO]", or conditional phrasing ("if supported by data"), treat these as signals to probe the author's reasoning, not as formatting issues.

5. **Fewer, sharper comments.** An outline review with 2-4 substantive comments is better than 8 variations of "this needs to be written." Avoid the "missing content" genre entirely when content is absent because it hasn't been written yet.

6. **When drafting from an outline:** Generate text that follows the bullet structure as a scaffold but produces real paragraph prose. Maintain the argument sequence the author planned.
